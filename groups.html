Working Groups

In accordance to the IHDEA mission as described in the IHDEA Charter, the work of the IHDEA can be organized into different working groups (WGs) that serve as focal points for various IHDEA activities designed to achieve its stated goals. To that end, the IHDEA has identified the following working groups:

  • WG1 on community awareness
    • Task is to ensure an effective IHDEA website and to define an approach to improve community awareness.
  • WG2 Community Metadata Assessment
    • Task is to define improvements to the SPASE information to enhance integration with IVOA/UCD and to extend the adoption of the SPASE information model by additional data centers (i.e. IUGONET, CEF, AMDA).
  • WG3 on Coordinate systems standardisation
    • Task is to describe coordinate systems in unambigous terms, especially where reference models differ over time [e.g. GSM (IGRF-12 vs IGRF-13)].
  • WG4 DOI on data
    • Task is to define the process for obtaining DATA DOI and the coordination of the task amoung the data centers.
  • WG5 on HAPI
    • Task is to improve the HAPI service and to expand adoption of of HAPI servers at more data centers.
  • WG6 on Metadata standard for simulations
    • Task is the implementation of Metadata standard to simulation models, simulations runs, and simulation output (CCMC and IMPEx). In particular, defining metadata for simulation models and generated data, as well as defining a standard for an execution interface.
  • WG7 to trace the usage of Python
    • Task is to quantify usage metrics for tools, services and web sites, especially the use of Python tools and access to data through web services.
  • WG8 on SPASE registry organisation and maintenance
    • Task is the organisation and maintenance of the metadata registries.
  • WG9 related to SPASE 3.0
    • Task is to define the next version of the SPASE information model.
  • WG10 on Standard for execution interface
    • Task is to define Standard for execution interface.
